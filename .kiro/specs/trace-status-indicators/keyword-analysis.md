# 关键词分析报告

## 实际使用的 log_auto() 调用统计

### 1. 成功类关键词（✅）
- "初始化" (7次) - LevelManager初始化、RoomBuilder初始化等
- "成功" (15次) - 资源加载成功、地图加载成功、PlayerStart类加载成功等
- "完成" (5次) - Level准备完成、训练室构建完成、光照系统设置完成等
- "创建" (7次) - 创建新地图、创建地板、创建天花板、创建墙壁等
- "生成" (2次) - 生成成功、生成PlayerStart
- "保存" (1次) - 保存地图文件
- "验证成功" (2次) - 地图文件验证成功、地图在资产系统中验证成功
- "放置成功" (1次) - PlayerStart放置成功
- "配置" (4次) - 配置方向光、配置天空光、设置GameMode覆盖等

### 2. 错误类关键词（❌）
- "错误：" (10次) - 错误：加载地图失败、错误：获取World失败、错误：创建Level失败等
- "失败" (6次) - 资源加载失败、保存地图失败、地图保存验证失败等

### 3. 警告类关键词（⚠️）
- "警告：" (4次) - 警告：加载DirectionalLight类失败、警告：加载SkyLight类失败、警告：获取World设置失败、警告：加载FPSTrainingGameMode类失败

### 4. 信息类（ℹ️，无特殊关键词）
- "开始" (5次) - 开始准备Level、开始构建训练室、开始放置PlayerStart等
- "加载" (2次) - 加载网格和材质、加载FPSTrainingGameMode类
- "获取" (2次) - 获取World引用、获取World设置
- "检查" (1次) - 检查现有地图文件
- "验证" (1次) - 验证资源加载
- "清理" (1次) - 清理旧的Actors
- "已清理" (1次) - 已清理 X 个旧Actors
- "设置" (1次) - 设置PlayerStart标签
- "计算" (1次) - 计算文件大小变化
- "地图已存在，加载中" (1次)

## 关键词优先级规则

1. **错误优先**：如果包含"错误"或"失败"，优先判定为错误（❌）
   - 例如："错误：资源加载失败" → ❌（而不是因为"加载"判定为信息）

2. **警告次之**：如果包含"警告"，判定为警告（⚠️）
   - 例如："警告：加载DirectionalLight类失败" → ⚠️

3. **成功第三**：如果包含成功关键词，判定为成功（✅）
   - 例如："资源加载成功" → ✅
   - 例如："训练室构建完成" → ✅

4. **默认信息**：其他情况默认为信息（ℹ️）
   - 例如："开始构建训练室" → ℹ️

## 推荐的关键词列表

### 成功关键词
```python
SUCCESS_KEYWORDS = [
    "成功", "完成", "创建", "生成", "保存", 
    "验证成功", "放置成功", "配置", "初始化"
]
```

### 错误关键词（优先级最高）
```python
ERROR_KEYWORDS = [
    "错误", "失败", "异常"
]
```

### 警告关键词
```python
WARNING_KEYWORDS = [
    "警告", "跳过", "未找到"
]
```

## 结论

**警告关键词确实有使用！** 在代码中有4处使用了"警告："前缀，主要用于非致命性错误（如可选组件加载失败）。

建议保留警告状态（⚠️），因为：
1. 实际代码中有使用（4次）
2. 区分"致命错误"和"非致命警告"很有价值
3. 未来可能会有更多警告场景
